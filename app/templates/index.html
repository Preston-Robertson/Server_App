    Authentication
    
        
    
        
    
          
    
            The bearer token lives in /etc/gamesrv.env
            on the LXC (GAMESRV_TOKEN=..., mode 660 root:gamesrv).
            Paste it once and this browser will remember it in
            localStorage.
          
    
          
    
            
            
            
            
            
          
    
        
    
      
    

      
    
        
    
          
    Manager Self-Update
    
        
    
        
    
          
    
            Update pulls the latest commit on the repo's default branch,
            reinstalls pinned Python requirements into the venv, restarts the manager,
            health-checks /healthz, and auto-rolls back on failure.
            Restart bounces the manager without pulling —
            handy after editing env values below (env is only read at startup).
          
    
          
    
            
            
            
            
          
    
          
    Log file: /opt/gamesrv/logs/update.log
    
          
    (no data yet — click Refresh log)
    
        
    
      
    

      
    
        
    
          
    Environment file
    
          -
        
    
        
    
          
    
            Managed keys have proper labels and validation; extra keys already
            in the file (per-server RCON passwords, git PATs, etc.) show up in
            their own section. Secrets left blank on save keep their current
            value — an empty submit won't wipe your token. After saving, hit
            Restart above so the new values take effect.
          
    
          
    
          
    
          
    
            
            
            
            
          
    
          
    
            Save Environment writes the file only. Env vars
            are re-read at process start, so anything you change (a new
            MC_*_GIT_PAT, a rotated GAMESRV_TOKEN)
            only takes effect after a restart. Use Save &
            Restart to do both in one shot.

            Access & Idle
            
    
              
              
            
    
            
            
            
    
              Native Steam ID enforcement works on ARK: SE / Ascended today.
              Other games fall back to password + IP allowlist.
            
    

            
    

            
    
              
              
            
    
            
    
              With wake-on-demand, the manager's proxy owns the public port and
              buffers client packets while the game starts — no "first attempt
              fails" for players. The game process is remapped to
              port + 10000 internally, so reinstall
              after toggling. Minecraft SLP status pings are
              answered locally so idle server-list refreshes don't wake the JVM.
            
    
          
    

          
    
            Game passwords (Palworld / ARK / Valheim)
            
    
              
              
            
    
            
    
              Passwords are written to servers/<name>.yml.
              Keep that directory out of any public git remote.
              Avoid spaces or the characters ? &
              " ' — those break the game's parser.
