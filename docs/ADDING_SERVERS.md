# Adding a new server

## 1. Pick a `type`

- `minecraft-java` — **Vanilla / Paper / Purpur / Fabric** (single `server.jar`
  launched with `java -jar`). Do **not** use this for modern Forge.
- `minecraft-forge` — **Modded Forge** on 1.17+ (yes, 1.20.1). Launches
  Forge's own `run.sh`, writes `-Xms/-Xmx` into `user_jvm_args.txt`
  (which is where Forge actually reads them), warns if `mc_version` and
  the system Java disagree, and runs everything inside tmux for the
  Console tab + graceful stop. Users upload the Forge server tree
  (`mods/`, `config/`, `libraries/`, `run.sh`, `user_jvm_args.txt`) via
  the Files tab or SFTP.
- `steamcmd` — anything installable via anonymous SteamCMD (Palworld, Valheim,
  ARK, 7DTD, etc.). Needs `steam_app_id`.
- `custom` — you write `start.sh` yourself. Use this for Bedrock, PZ,
  Terraria, Fabric with a non-standard launcher, or anything not covered
  above.

## 2. Definition schema

`servers/<name>.yml`:

```yaml
name: myserver                       # [a-z][a-z0-9-]{1,31}, must match filename stem
type: minecraft-java                 # minecraft-java | steamcmd | custom
install_dir: /srv/gameservers/myserver
world_dir: /opt/gamesrv/worlds/myserver   # on the TrueNAS bind mount
port: 25565
memory_mb: 4096                      # -Xmx for Java; informational otherwise
java_args: ""                        # minecraft-java only
start_cmd: "./start.sh"              # regenerated by install for minecraft/steamcmd
stop_cmd: ""                         # console command sent before shutdown (minecraft: "stop")
auto_start_on_boot: true

# steamcmd only:
steam_app_id: 0
steam_beta: ""                       # optional public-beta branch name

# minecraft-forge only (informational; surfaced on dashboard):
mc_version: ""                       # e.g. "1.20.1" — enables the Java-version sanity check
forge_version: ""                    # e.g. "47.3.0"

extra_env: {}                        # extra KEY=VAL written to install_dir/server.env

rcon:
  enabled: false
  port: 25575
  password_env: ""                   # NAME of env var in /etc/gamesrv.env; never inline pw

backup:
  enabled: true
  target: ""                         # blank => world_dir
```

## 3. Workflow

1. Create the YAML (or use the dashboard's **+ New Server**).
2. **Control → Install** to provision.
3. Upload jars / configs (**Files** tab).
4. **Control → Start**.
5. **Control → Enable on Boot** once you're happy.

## 4. Adding a new `steamcmd` app not in the built-in recipes

Edit `app/types/steamcmd.py`, add to `_APP_RECIPES`:

```python
_APP_RECIPES[123456] = {
    "name": "mygame",
    "run": "./MyGameServer -port={port} -someflag",
    "saves_rel": "Saved/SaveGames",   # relative to install_dir; symlinked to world_dir
}
```

Restart the manager. Set `type: steamcmd`, `steam_app_id: 123456` in your YAML,
then **Install**.

## 5. Converting a `.bat` launcher

```bash
python3 /opt/gamesrv/scripts/convert_bat_to_sh.py /path/to/start.bat > /tmp/start.sh
# Review /tmp/start.sh, wrap the launch line in a tmux session, then upload as install/start.sh
```

Minimum tmux wrapper (for the console + graceful stop features to work):

```bash
#!/usr/bin/env bash
set -euo pipefail
cd "$(dirname "$0")"
tmux kill-session -t "gs-myserver" 2>/dev/null || true
exec tmux new-session -s "gs-myserver" -n game "./MyGameServer -port=12345"
```

## 6. Modded Minecraft (Forge) specifics

Use `type: minecraft-forge`. The handler assumes the standard Forge 1.17+
tree in `install_dir`:

```
install_dir/
  run.sh                       # from the Forge installer's --installServer output
  user_jvm_args.txt            # REGENERATED by the handler on Install/Update
  libraries/net/minecraftforge/forge/<mc>-<forge>/unix_args.txt
  mods/
  config/
  defaultconfigs/
  server.properties
  eula.txt                     # must contain eula=true before first start
  whitelist.json ops.json      # optional
world_dir/                     # bind mount to TrueNAS; symlinked in as install_dir/world
```

The handler will:

- Regenerate `start.sh` (launches `./run.sh nogui` inside tmux),
  `stop.sh` (sends `stop` + waits up to 120 s), and `user_jvm_args.txt`
  (with `-Xms` / `-Xmx` from `memory_mb`) every time you click **Install**.
- Symlink `install_dir/world` \u2192 `world_dir`.
- Warn you if `mc_version` is set and the system `java` version doesn't
  match Mojang's requirement (`1.20.x` \u2192 Java 17; `1.21.x` \u2192 Java 21).
  Forge 1.20.1 will **not** run on Java 21 \u2014 don't upgrade for this pack.
- Refuse to auto-accept the EULA \u2014 you must edit `eula.txt`.

**Updating the game software:** *manual*. Upload replacement mod jars into
`mods/` and/or a new Forge tree, then click **Update Game Software**. There\u2019s\nno reliable auto-update for a modded pack, so the handler just re-runs the\nprovisioning step (regenerates scripts + `user_jvm_args.txt` in case\n`memory_mb` changed).

**Do NOT use `type: minecraft-java` for Forge.** That handler runs\n`java -jar server.jar nogui`, which is the wrong entrypoint for modern\nForge \u2014 it won\u2019t start.
